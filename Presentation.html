Smart, Per-Repository Guidelines

📄 Repo Guidelines File

Add a CODE_REVIEW_GUIDELINES.md to any repository's main branch and the bot uses it as primary review rules.

# CODE_REVIEW_GUIDELINES.md ## Our Rules - All public APIs must be documented - No raw SQL — use EF Core only - Services must be scoped lifetime - Max method length: 40 lines - React components must use hooks

🔍 Intelligent Fallback

No guidelines file? The bot analyses the changed file extensions and automatically applies language-specific rules.

  • .cs → .NET/C# rules (async, null safety, EF)
  • .ts/.tsx → TypeScript/React rules (hooks, types)
  • .py → Python rules (mutable defaults, SQL)
  • .go → Go rules (errors, goroutine leaks)
  • .sql → SQL rules (injection, indexes)

Branch-Based Review Policy

Only review PRs that matter — ignore noise from internal branches.

✅ Source Branch Patterns

Wildcards supported — only these branches trigger a review:

feature/* bugfix/* hotfix/* fix/* release/*

🎯 Target Branches

Only PRs merging into protected branches get reviewed:

main master develop

All patterns are configurable in appsettings.json — no redeployment needed.

Architecture

📡

Webhook Receiver

Rate-limited endpoint validates HMAC signature, deduplicates by commit hash, and enqueues the payload.

⚙️

Background Queue

Bounded channel processes one PR at a time. Back-pressure returns 503 if the queue is full. Graceful shutdown drains cleanly.

🔍

Diff Parser

Parses unified git diffs into structured hunks with accurate new-file line numbers per file.

🧠

Claude Integration

Direct Anthropic Messages API. Prompt template is a hot-reloadable file — edit it without restarting.

Retry + Circuit Breaker Health Checks Structured Logging (Serilog) Security Headers
